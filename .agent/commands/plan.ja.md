# /plan コマンド

機能や変更のための実装計画を作成します。

## 概要

Clean Architecture の原則に従って、包括的な実装計画を作成します。

## 使い方

```
/plan <機能の説明>
```

## プロセス

1. 機能要件を分析する
2. 影響を受ける Clean Architecture のレイヤーを特定する
3. 依存関係を含むタスクの細分化を行う
4. テスト要件を定義する
5. 必要なドキュメントの更新をリストアップする

## 出力

レビュー可能な Markdown 形式で構成された実装計画。
