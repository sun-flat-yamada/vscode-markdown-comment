# Documentation Rules

変更時のドキュメント更新に関する必須ルールを定義します。

## 必須ルール

> [!IMPORTANT]
> **変更を行った際は、必ず影響ドキュメントを総点検して更新反映すること。**

これはプロジェクトの厳格な要件であり、例外は認められません。

## ドキュメント更新チェックリスト

### 変更後に確認すべき項目

1. **README.md**
   - [ ] 機能説明が最新か
   - [ ] 使用方法が正確か
   - [ ] インストール手順が有効か

2. **GEMINI.md**
   - [ ] アーキテクチャ図が現状を反映しているか
   - [ ] クイックリファレンスが正確か
   - [ ] AI 向けオンボーディング情報が最新か

3. **インラインコメント**
   - [ ] 複雑なロジックに説明があるか
   - [ ] 古いコメントが削除されているか

4. **ワークフロードキュメント**
   - [ ] `.agent/workflows/` の手順が有効か

## 英語マスター原則

- **英語が正本**: 英語 `.md` ファイルのみを参照・編集する
- **日本語は参照専用**: `.ja.md` ファイルは読まない・編集しない

## 自動リマインダー

`hooks.json` に定義されたフックにより、変更時に自動的にドキュメント更新がリマインドされます。
