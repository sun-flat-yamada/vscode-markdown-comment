# Documentation Rules

変更時のドキュメント更新に関する必須ルールを定義します。

## 必須ルール

> [!IMPORTANT]
> **変更を行った際は、必ず影響ドキュメントを総点検して更新反映すること。**

これはプロジェクトの厳格な要件であり、例外は認められません。

## ドキュメント更新チェックリスト

### 変更後に確認すべき項目

1. **README.md**
   - [ ] 機能説明が最新か
   - [ ] 使用方法が正確か
   - [ ] インストール手順が有効か

2. **GEMINI.md**
   - [ ] アーキテクチャ図が現状を反映しているか
   - [ ] クイックリファレンスが正確か
   - [ ] AI 向けオンボーディング情報が最新か

3. **インラインコメント**
   - [ ] 複雑なロジックに説明があるか
   - [ ] 古いコメントが削除されているか

4. **ワークフロードキュメント**
   - [ ] `.agent/workflows/` の手順が有効か

## 英語マスター原則

- **英語が正本**: 英語 `.md` ファイルのみを参照・編集する
- **日本語は参照専用**: `.ja.md` ファイルは読まない・編集しない

## 即時翻訳原則 (Immediate Translation)

- **エージェントの責務**: 英語ドキュメント (`*.md`) を変更または新規作成した場合、**同ターン内または一連のタスク内**で、必ず対応する日本語版 (`*.ja.md`) を翻訳・生成してください。
- **後回し禁止**: 自動化スクリプトはあくまで「漏れ検知」用です。翻訳作業をスクリプト任せにしたり、後回しにしてはいけません。
- **完全同期**: 日本語版の内容は、英語版と情報の粒度・構成を完全に一致させてください。

## 自動リマインダー

`hooks.json` に定義されたフックにより、変更時に自動的にドキュメント更新がリマインドされます。
